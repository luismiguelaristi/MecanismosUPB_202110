Análisis de posición del mecanismo manivela corredera por métodos numéricos¶

Objetivo¶

Encontrar valores de las variables del mecanismo que me permitan posicionar sus barras, usando métodos numéricos.

Diagramas cinemático y vectorial¶

Usar este enlace para escribir las ecuaciones en formáto LaTeX: https://codecogs.com/latex/eqneditor.php

Parámetros: $r2$, $r3$, $\theta_4$

Variable de entrada: $\theta_2$

Variables secundarias: $\theta_3$ y $r_4$

A partir del diagrama vectorial, se puede plantear una ecuación vectorial que represente al mecanismo:

$\overrightarrow{r_{2}}-\overrightarrow{r_{3}}-\overrightarrow{r_{4}} = 0$ (1)

cuyas ecuaciones escalares son

$r2\cdot cos(\theta_{2})-r3\cdot cos(\theta_{3})-r4\cdot cos(\theta_{4})=0$, (2)

$r2\cdot sin(\theta_{2})-r3\cdot sin(\theta_{3})-r4\cdot sin(\theta_{4})=0$ (3).

El problema ahora consiste en resolver este sistema de ecuaciones NO LINEAL. Para esto usaremos el módulo de optimización de Scipy, específicamente la función fsolve.

Solución¶

Primero importamos los módulos necesarios:

  1. numpy para operaciones numéricas
  2. fsolve de scipy.optimize para resolver el sistema de ecuaciones no lineal
In [252]:
import numpy as np
from scipy.optimize import fsolve

Para resolver un sistema de ecuaciones NO LINEAL usando el método numérico de Newton-Rhapson con "fsolve", debemos proveer una FUNCIÓN.

Repaso sobre funciones¶

Una función es un bloque de código reutilizable con entradas y una salida. Se pueden procesar los datos de entrada para generar la salida.

Definamos esta función que haga una simple suma:

In [253]:
def funcion(in_1,in_2):
    out = in_1 + in_2
    return out

Si tratamos de ejecutar la función, no pasa nada, ya que lo que acabamos de hacer es definir la función, pero aún no la hemos utilizado.

Ahora para utilizarla:

In [254]:
resultado = funcion(2,2)
print(resultado)

4

Las entradas de la función se definen en el paréntesis, la salida de la función se obtiene al ponerla al lado derecho de la igualdad, en este caso queda almacenada en la variable "resultado".

Lo interesante de las funciones es que son reutilizables.

Usando fsolve¶

Para usar fsolve, debemos incluir el sistema de ecuaciones en una función de la siguiente forma:

In [255]:
def manivelaCorredera(condIni, params):
    #variable de entrada y parámetros
    ## Primero la variable de entrada
    th2 = params[0]
    ## Luego los parámetros
    r2 = params[1]
    r3 = params[2]
    th4 = params[3]
    
    # las condiciones iniciales deben darse para que el método funcione
    th3 = condIni[0]
    r4 = condIni[1]
    
    # creamos el vector donde quedará la salida, inicialmente con ceros
    Y = np.zeros(2)
    # Definimos las ecuaciones escalares no lineales
    Y[0] = r2*np.cos(th2) - r3*np.cos(th3) - r4*np.cos(th4);
    Y[1] = r2*np.sin(th2) - r3*np.sin(th3) - r4*np.sin(th4);
    
    return Y

Probar la función¶

Podemos probar la función simplemente definiendo los parámetros correspondientes al mecanismo, un valor para la variable de entrada y las condiciones iniciales:

In [256]:
r2 = 5
r3 = 17.5
th4 = 0
print("los parámetros son: ")
print(r2,r3,th4)

th2 = np.deg2rad(0)

los parámetros son: 
5 17.5 0

ahora para las condiciones iniciales, son valores aproximados de las incógnitas que debemos proveer como punto de partida para que fsolve inicie la búsqueda iterativa de la respuesta.

En este caso, podemos usar los valores exactos que obtuvimos con el método analítico, los cuales deberían hacer que las ecuaciones escalares se cumplan, es decir

Y[0] = $r2cos(\theta_{2})-r3cos(\theta_{3})-r4cos(\theta_{4})=0$

Y[1] = $r2sin(\theta_{2})-r3sin(\theta_{3})-r4sin(\theta_{4})=0$

que para $\theta_2 =$

In [257]:
print(np.rad2deg(th2))

0.0

$r_4$ [cm] y $\theta_3$ [°] corresponden respectivamente a

In [258]:
th3ini = np.deg2rad(180)
r4ini = 22.5

Por lo que si llamamos a la función

In [259]:
x0 = [th3ini,r4ini]
parametros = [th2,r2,r3,th4]
print(manivelaCorredera(x0,parametros))

[ 0.0000000e+00 -2.1431319e-15]

Comprobamos que se cumplen las ecuaciones escalares para los valores de $r_4$ y $\theta_3$ propuestos.

Usualmente no vamos a tener valores exactos para las condiciones iniciales, por lo que podemos usar valores aproximados obtenidos por método gráfico.

ATENCIÓN¶

Si en este paso obtenemos valores muy diferentes a cero (con valores enteros mayores o menores a cero), debemos revisar varias cosas:

  • las condiciones iniciales están muy alejadas de la realidad del mecanismo,
  • las ecuaciones escalares están mal planteadas,
  • los parámetros o la variable de entrada no corresponden con la posición analizada del mecanismo.

Resolver esto antes de continuar.

Probar fsolve¶

Probemos fsolve de la siguiente forma

In [260]:
q = fsolve(manivelaCorredera,x0, args = parametros)
print("Theta 3 = ",np.rad2deg(q[0])," y r4 = ",q[1])

Theta 3 =  180.0  y r4 =  22.5

Observemos que usando fsolve obtenemos lo que queremos hallar, es decir, los valores de las incógnitas. Sin embargo, en este caso no tiene mucha gracia que hayamos obtenido lo que ya conocíamos. Lo interesante viene después:

  • conociendo una posición inicial del mecanismo, podemos hallar otras posiciones cercanas, por ejemplo, si movemos la manivela unos cuantos grados, por ejemplo
In [261]:
incremento = 10 # grados
th2 = th2 + np.deg2rad(incremento)
# actualizamos el valor de la variable de entrada en el vector de parámetros
parametros[0] = th2

Ahora volvamos a resolver usando como condiciones iniciales el último resultado después de usar fsolve

In [262]:
x0 = q
q = fsolve(manivelaCorredera,x0, args = parametros)
th3 = q[0]
r4 = q[1]
print("Theta 3 = ",np.rad2deg(th3)," y r4 = ",r4)

Theta 3 =  177.156173150156  y r4 =  22.40248714470529

Ahora si, vamos a implementar un ciclo for para las "numpos" posiciones de la manivela

In [268]:
for i in range(0,numpos):
    # para cada iteración actualizamos el valor de la variable de entrada
    parametros[0] = th2V[i]
    # resolvemos con fsolve
    q,info,exitflagV[i],mensaje = fsolve(manivelaCorredera,
                                        x0, 
                                        args = parametros,
                                        full_output = True)
    th3V[i] = q[0]
    r4V[i] = q[1]
    x0 = q
    # Opcional, poner esto para detectar si hay un error
    if exitflagV[i] != 1:
        print('Error en la iteración ' + str(i))
        print(mensaje)
        break

Preparación de animación¶

Para hacer animaciones haremos uso del módulo animation de matplotlib.

In [274]:
from matplotlib.animation import FuncAnimation

Este módulo requiere también una función donde indiquemos qué se requiere animar, de esta forma:

In [275]:
def animar(i):
    manivelaX = np.array([Ox,AxV[i]])
    manivelaY = np.array([Oy,AyV[i]])
    acopladorX = np.array([BxV[i],AxV[i]])
    acopladorY = np.array([ByV[i],AyV[i]])
    hr2[0].set_xdata(manivelaX)
    hr2[0].set_ydata(manivelaY)
    hr3[0].set_xdata(acopladorX)
    hr3[0].set_ydata(acopladorY)

Teniendo la función se puede realizar la animación:

In [276]:
duracion = 3000 # en milisegundos
animacion = FuncAnimation(fig, animar, interval=duracion/numpos, save_count=numpos)
